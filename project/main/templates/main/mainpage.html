ch1 HTML 페이지

templates 폴더 생성

  • 왜 만드는가? : 유저에게 제공할 파일의 구조나 레이아웃을 한 데 모아 관리하기 위함 (ex, HTML, CSS)
  • main 앱 내에 templates 폴더를 생성
  • templates 폴더 안에는 앱 이름과 동일한 폴더를 생성
  • (pagename).html 파일 생성 후 코드 작성

ch2 HTML 페이지와 View 연결

views.py 작성

  • 원하는 url을 통해 방금 작성한 (pagename)가 보여지게 하기 위해 views.py를 통해 pagename가 렌더링 될 수 있도록 작성한다.
  • url을 입력한다.
  • url에 해당하는 데이터를 요청한다 (request)
  • 데이터에 맞는 html을 전송한다 (render)
  • 화면에 보여준다

urls.py작성

  • url에 해당하는 데이터 연결하기
  • main 앱에서 views 함수를 불러옴
  • path(’ ‘, views.mainpage, name="mainpage")
  • 앞서 작성한 함수를 연결한다
  • url의 별명

템플릿 언어

  • 파이썬의 변수, 문법을 HTML에서 사용할 수 있도록 제공하는 직관적인 언어
  • 템플릿 필터 : 템플릿 변수를 특정 서식에 따라 변환하는 것 → 변수명 | 필터
  • 템플릿 주석 : HTML 내용 중 주석(숨김)이 필요한 경우 사용 → {# 주석 #}

Templates 상속

  • 반복되는 코드 상속하기
  • HTML에서 반복적으로 사용되는 코드를 따로 분리하여, 다른 페이지에서도 사용 가능하도록 한다.

Static

  • 웹 페이지에서 사용되는 정적인 파일을 저장하는 디렉토리
  • image, JS, CSS
  • HTML 파일에서 활용 경로 설정 : settings.py
